# 🌿 Módulo 04: Control de Versiones con Git

Un desarrollador no trabaja solo en archivos; trabaja en equipo y necesita una "máquina del tiempo" para su código.

## 📍 Temas a Dominar

### 1. Conceptos de Git
- **Flujo Básico**: `git init`, `git add`, `git commit`.
- **Ramas (Branches)**: Por qué nunca deberías trabajar directamente en `main`. `git branch`, `git checkout -b`.
- **Sincronización**: `git push`, `git pull`, `git fetch`.

### 2. GitHub / Lab
- **Repositorios Remotos**: Conectar tu código local con la nube.
- **Pull Requests (PR)**: Cómo proponer cambios y revisar el código de otros.
- **Issues y Tableros**: Organizar tu trabajo como un profesional.

## 🛠️ Retos Sugeridos
1.  Crear tu primer repositorio, subirlo y compartirlo con alguien más.
2.  Simular un conflicto de código (Merge Conflict) y aprender a resolverlo.
